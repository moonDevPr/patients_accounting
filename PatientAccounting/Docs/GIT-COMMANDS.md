## Работа с GIT
небольшой файл-подсказка для участников проекта

## Ветки
1 вариант создания:
- создание ветки
```bash
git branch <название ветки>
```

- переключение на ветку
```bash
git checkout <название ветки>
```

2 вариант:
```bash
git checkout -b <название_ветки>
```

- переключение между ветками
```bash
git switch <название_ветки>
```

### Обыденная ситуация (не путаемся)
- Программист Леша создал ветку, закомитил и отправил изменения на свою ветку (в какой-то момент времени), код слили в main. 
- Другой программист Саша создал новую ветку, повторил действия 1 программиста. 
- Программист Леша вдруг решил поменять свой код в своей ветке. Но его ветка устарела и нужно актуализировать данные, взяв их из главной ветки. Иначе мы получим море конфликтов

#### Решение:
1. Проверяем, что мы точно в нашей ветке:
```bash
git checkout <название_ветки>
```
2. Получаем последние изменения из основной ветки
```bash
git fetch origin main
```
3. Объединяем изменения из основной ветки с рабочей веткой
```bash
git merge origin/main
```
4. Теперь, когда ветка обновлена с учётом последних изменений в основной ветке, можно отправить свои изменения в удалённую ветку
```bash
git push origin <название ветки>
```
